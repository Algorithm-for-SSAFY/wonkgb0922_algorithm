### 문제 설명

<p>크기가 N(1 ≤ N ≤ 100,000)인 1차원 배열 A[1], …, A[N]이 있다. 어떤 i, j(1 ≤ i ≤ j ≤ N)에 대한 점수는, (A[i] + … + A[j]) × min{A[i], …, A[j]}가 된다. 즉, i부터 j까지의 합에 i부터 j까지의 최솟값을 곱한 것이 점수가 된다.</p>

<p>배열이 주어졌을 때, 최대의 점수를 갖는 부분배열을 골라내는 프로그램을 작성하시오.</p>

### 입력 

 <p>첫째 줄에 정수 N이 주어진다. 다음 줄에는 A[1], …, A[N]을 나타내는 정수들이 주어진다. 각각의 정수들은 음이 아닌 값을 가지며, 1,000,000을 넘지 않는다.</p>

### 출력 

 <p>첫째 줄에 최대 점수를 출력하고, 둘째 줄에 그 구간의 시작 위치(i)와 끝 위치(j)를 출력한다.</p>
 
### 💡접근 방법
> 어떤 구간 [i, j)(i <= j)를 선택했다고 하자. 여기서 구간 [i ,j)의 합은 누적합으로 O(1)만에 얻을 수 있고, 구간의 최솟값은 세그먼트 트리를 이용하여 O(logN)만에 얻을 수 있다. 여기서 구간은 일반적으로 선택하면 O(n^2)이지만, 이 문제의 핵심은 최소값을 곱하는 데 있다. 즉, 최솟값을 포함한다면 최대한 넓은 구간을 산정하는 것이 유리하다.
> 초기에 구간 [0, n)으로 시작하고 해당 구간의 점수를 계산한다. 그리고 해당 구간 내의 최솟값을 제외하고 나머지 구간을 계산한다.
> 즉, [0, n) 안의 최솟값 위치를 `mid`라고 했을 때, [0, mid), [mid + 1, n)의 점수를 계산하면 된다. 이게 가능한 것은, 어떠한 구간에서 최솟값을 포함할 때, 그 최솟값보다 작은 값이 나오지 않는 구간으로, 최대로 잡는 것이 유리하기 때문이다. 이렇게 구간마저 O(logN)으로 최적화할 수 있다.
> 총 시간복잡도는 아마도 O(logN*(N + logN)) 정도가 될 듯하다.
>
> 세그먼트 트리는 바텀업 방식으로 구현하였다.
